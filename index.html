Prática Autoguiada: O Herói da Lógica

Modo Treino Ativado: A cópia de texto está bloqueada. Para aprender, você terá que ler e digitar o código manualmente!

Bem-vindo ao seu teste de fixação! Hoje você vai aplicar os 4 Pilares do Pensamento Computacional (Decomposição, Padrões, Abstração e Algoritmos) criando um pequeno RPG de texto passo a passo no console do seu navegador.

1

Preparação: Abrir o Laboratório

Para que serve este passo? Antes de programarmos, precisamos acessar o "cérebro" do navegador. Esta é a nossa área de trabalho.

Abra uma nova aba no seu navegador (Chrome, Edge, etc.).

Pressione a tecla F12 no seu teclado (ou Ctrl + Shift + J). Clique na aba que diz "Console". É aqui que você vai digitar todos os comandos a seguir.

2

Abstração: Criando o Herói

Para que serve este passo? Usamos a Abstração para guardar apenas a informação essencial na memória (Variáveis). Não importa a cor do cabelo do herói, apenas o seu nome e energia.

Digite o código abaixo no seu console e pressione ENTER após cada linha:

var nomeHeroi = "O seu Nome Aqui";
var energia = 100;
3

Decomposição: O Dano do Monstro

Para que serve este passo? Estamos dividindo um problema (o combate) em uma operação matemática simples que o computador consegue resolver facilmente.

O seu herói levou um golpe! Vamos criar uma variável para o dano e subtrair da energia total. Digite:

var dano = 30;
energia = energia - dano;

4

Algoritmo (Saída): O Ponto de Situação

Para que serve este passo? Os dados foram processados, mas precisamos que o algoritmo mostre o resultado para o jogador usando um alerta.

Vamos juntar os textos (Strings) com as variáveis usando o sinal de +. Digite e aperte ENTER para ver o aviso na tela:

alert("Atenção " + nomeHeroi + "! Sua energia desceu para: " + energia);
5

Interação: A Escolha do Jogador

Para que serve este passo? Um bom algoritmo interage com o usuário. Usamos o comando "prompt" para receber uma resposta e guardá-la em uma variável.

O jogo precisa de uma decisão sua. Digite o comando abaixo, e quando a caixa aparecer, escreva sim ou nao.

var escolha = prompt("Deseja beber uma poção mágica? (sim/nao)");
6

Padrões (Arrays): O Inventário

Para que serve este passo? Quando temos vários itens semelhantes, o padrão nos diz para agrupá-los em uma Lista (Array) em vez de criar dezenas de variáveis separadas.

Vamos verificar o que você tem na mochila. Digite a seguinte lista (use os colchetes):

var inventario = ["Mapa", "Corda", "Bússola"];
7

Alterando a Lista (Push)

Para que serve este passo? As listas não são fixas. Usamos ferramentas específicas da linguagem (neste caso, "push") para adicionar dados no final do padrão criado.

Você encontrou uma chave escondida! Adicione-a ao inventário e veja o resultado no console:

inventario.push("Chave de Ouro");
console.log(inventario);
Página 2
8

Decisão Lógica: IF e ELSE

Para que serve este passo? O núcleo da programação. O computador avalia a sua "escolha" (Passo 5) e decide qual caminho seguir com base em uma condição verdadeira ou falsa.

Escreva a estrutura abaixo exatamente com os parênteses e chaves. Cuidado com o duplo igual == que serve para comparar coisas!

if (escolha == "sim") {
    energia = energia + 50;
    alert("Você bebeu a poção! Energia recuperada: " + energia);
} else {
    alert("Você guardou a poção. Energia continua sendo: " + energia);
}
9

Compreendendo o Seu Algoritmo

Para que serve este passo? O pensamento computacional exige validação. Você precisa olhar para tudo o que escreveu e entender o fluxo completo da história.

Se você fez tudo corretamente até aqui, você construiu um Algoritmo Completo. Reflita sobre isso:

  • Você definiu os dados de entrada (Nome, Energia, Inventário).
  • Você processou informações (Dano, Escolha do usuário).
  • Você estabeleceu regras lógicas (If/Else) para alterar o resultado.
  • Você gerou uma saída (Alert).
