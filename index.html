Treinamento: Variáveis

Construção de Memória Muscular na Digitação de Código

Bloqueio de Cópia Ativado: Para dominar as variáveis, você precisa digitar. Construa os 10 passos sequencialmente!

Abra o seu ambiente de desenvolvimento clicando abaixo:

Acessar Laboratório Virtual

Fase 1: Os Tipos Primitivos

Inicie o seu código no Laboratório Virtual declarando os dados básicos do seu personagem usando let. Não apague os códigos de um passo para o outro!

1

Criando uma String (Texto)

As Strings armazenam textos e sempre exigem o uso de aspas.
// 1. Nome do Jogador
let nomeJogador = "Comandante Shepard";
2

Criando um Number (Número)

Numbers não levam aspas. Eles servem para realizar cálculos matemáticos depois.
// 2. Nível Inicial
let nivel = 1;
let moedas = 50;
3

Criando um Boolean (Lógico)

Booleans representam estados binários (Verdadeiro ou Falso). Digite sem aspas.
// 3. Status do Escudo
let escudoAtivo = true;

Fase 2: Manipulando a Memória

As variáveis podem sofrer alterações e interagir umas com as outras.

4

Atualizando um Valor (Reatribuição)

Quando a "caixa" (variável) já foi criada com let, não precisamos usar a palavra let novamente para trocar o que tem dentro dela.

Seu personagem subiu de nível! Atualize o número na memória:

// 4. Subindo de Nível (não use 'let' aqui!)
nivel = 2;
5

Operações Matemáticas Simples

O computador resolve o cálculo matemático à direita do sinal de = e guarda o resultado final na variável à esquerda.

Você encontrou um baú de tesouro com 30 moedas extras. Vamos somar!

// 5. Somando moedas
let bonus = 30;
moedas = moedas + bonus;
6

Juntando Textos (Concatenação)

Quando usamos o sinal de + junto com uma String, o computador para de fazer matemática e "cola" os textos lado a lado.

Crie uma mensagem dinâmica que muda de acordo com o nome do jogador:

// 6. Mensagem de Boas-Vindas
let saudacao = "Bem-vindo de volta, " + nomeJogador + "!";
alert(saudacao);
Página 2

Fase 3: Entrada e Saída (I/O)

A verdadeira utilidade das variáveis é armazenar dados que vêm do mundo exterior (do usuário).

7

Recebendo Dados (Prompt)

O prompt abre uma caixa de pergunta. O que o usuário digitar será capturado pelo sinal de = e jogado dentro da nossa variável.

O sistema precisa da sua confirmação de qual classe você deseja jogar (Ex: Mago, Guerreiro):

// 7. Escolhendo a Classe
let classe = prompt("Qual classe você escolhe?");
8

Verificação Silenciosa (Console.log)

O console.log() não mostra janelas na tela. Ele envia mensagens discretas para a aba "Console" do desenvolvedor. Excelente para caçar bugs!

Mande um log das suas variáveis ativas para a máquina registrar:

// 8. Inspecionando variáveis no F12
console.log("Classe escolhida:", classe);
console.log("Escudo está ativo?", escudoAtivo);
9

Unindo Múltiplas Variáveis

Você pode concatenar (juntar com +) várias variáveis e vários textos de uma vez só!

Crie a mensagem do perfil que junta todos os seus dados:

// 9. O Perfil Completo
let perfil = nomeJogador + " | Nível: " + nivel + " | Moedas: " + moedas;
Página 3

Fase 4: O Relatório Final

10

Saída de Dados Completa

O último comando executa tudo. O \n dentro do texto é um truque ninja que significa "Nova Linha" (como dar um ENTER no texto).

Para encerrar o seu programa, dispare o alerta final que exibe o seu Perfil e a sua Classe:

// 10. O Resultado Final
alert("=== SISTEMA INICIADO ===\n" + perfil + "\nClasse de Combate: " + classe);

Executando a Missão!

O seu código tem 10 passos. Agora, no Laboratório Virtual, faça o seguinte:

  • Clique no botão azul "Run" (Executar).
  • O sistema vai mostrar o seu "Bem-vindo de volta". Clique em OK.
  • A seguir, ele vai perguntar a sua classe (prompt). Escreva "Guerreiro" ou "Mago" e dê OK.
  • Por fim, a janela do Relatório (Passo 10) deve aparecer mostrando que você está no Nível 2 e possui 80 Moedas.

Tudo funcionou? Copie todo o código do seu laboratório (Ctrl+A e Ctrl+C) e cole na atividade do PDA para garantir a sua nota!
